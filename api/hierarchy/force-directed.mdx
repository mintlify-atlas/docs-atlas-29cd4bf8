## Overview

The `ForceDirected` class creates a force-directed network diagram where nodes are positioned using physics-based simulation. Nodes attract or repel each other based on configured forces.

```typescript
import * as am5hierarchy from "@amcharts/amcharts5/hierarchy";

const forceDirected = root.container.children.push(
  am5hierarchy.ForceDirected.new(root, {
    centerStrength: 0.5,
    manyBodyStrength: -15
  })
);
```

- Series

## Settings

### nodePadding

<ParamField path="nodePadding" type="number">
  Minimum gap in pixels between the nodes.
</ParamField>

```javascript
forceDirected.set("nodePadding", 10);
```

### centerStrength

<ParamField path="centerStrength" type="number" default="0.5">
  A force that attracts (or pushes back) all nodes to the center of the chart.
  
  - Positive values attract nodes to center
  - Negative values push nodes away from center
  - Higher absolute values = stronger force
</ParamField>

```javascript
forceDirected.set("centerStrength", 1.2);
```

### manyBodyStrength

<ParamField path="manyBodyStrength" type="number" default="-15">
  A force that attracts (or pushes back) all nodes to each other.
  
  - Negative values make nodes repel each other (typical)
  - Positive values make nodes attract each other
  - Higher absolute values = stronger force
</ParamField>

```javascript
forceDirected.set("manyBodyStrength", -20);
```

### linkWithStrength

<ParamField path="linkWithStrength" type="number" default="0.5">
  A force that attracts (or pushes back) nodes that are linked together via `linkWithField`.
  
  Controls the strength of custom links between non-hierarchical nodes.
</ParamField>

```javascript
forceDirected.set("linkWithStrength", 0.8);
```

### velocityDecay

<ParamField path="velocityDecay" type="number" default="0.5">
  Resistance acting against node speed.
  
  The greater the value, the more "sluggish" the nodes will be.
  
  - Range: 0 to 1
  - Higher values = more friction/slower movement
</ParamField>

```javascript
forceDirected.set("velocityDecay", 0.7);
```

### initialFrames

<ParamField path="initialFrames" type="number" default="500">
  Length of how long initial force simulation would run in frames.
  
  The simulation runs for this many frames to settle nodes into position before displaying.
</ParamField>

```javascript
forceDirected.set("initialFrames", 300);
```

### showOnFrame

<ParamField path="showOnFrame" type="number" default="10">
  If set to a number, will wait X number of frames before revealing the tree.
  
  Can be used to hide initial animations where nodes settle into their places.
</ParamField>

```javascript
forceDirected.set("showOnFrame", 50);
```

### minRadius

<ParamField path="minRadius" type="number | Percent" default="1%">
  Smallest possible radius for a node circle.
  
  Can be a fixed pixel value or percent relative to chart size.
</ParamField>

```javascript
forceDirected.set("minRadius", am5.percent(2));
forceDirected.set("minRadius", 15); // 15 pixels
```

### maxRadius

<ParamField path="maxRadius" type="number | Percent" default="8%">
  Biggest possible radius for a node circle.
  
  Can be a fixed pixel value or percent relative to chart size.
</ParamField>

```javascript
forceDirected.set("maxRadius", am5.percent(10));
forceDirected.set("maxRadius", 50); // 50 pixels
```

### xField

<ParamField path="xField" type="string">
  Field in data that holds X coordinate of the node.
  
  Use to position specific nodes at fixed locations.
</ParamField>

```javascript
forceDirected.set("xField", "x");
```

### yField

<ParamField path="yField" type="string">
  Field in data that holds Y coordinate of the node.
  
  Use to position specific nodes at fixed locations.
</ParamField>

```javascript
forceDirected.set("yField", "y");
```

### Inherited Settings

ForceDirected inherits all settings from LinkedHierarchy and Hierarchy, including:

- `sort` - How to sort nodes by value ("ascending" | "descending" | "none")
- `valueField` - Field in data holding numeric value
- `categoryField` - Field in data holding category name
- `childDataField` - Field in data holding array of children
- `downDepth` - Number of child levels to show when drilling down
- `upDepth` - Number of parent levels to show from selected node
- `initialDepth` - Number of levels to show on first load
- `topDepth` - Starting level to show (hides upper levels)
- `singleBranchOnly` - Collapse other branches when one expands
- `animationDuration` - Duration for drill animations in milliseconds
- `animationEasing` - Easing function for animations
- `colors` - ColorSet for auto-assigning node colors
- `patterns` - PatternSet for auto-assigning node patterns
- `linkWithField` - Field holding IDs of nodes to link with

## Data Item Properties

Each data item in a ForceDirected chart has the following properties:

### children

<ResponseField name="children" type="Array<DataItem<IForceDirectedDataItem>>">
  An array of data items of child nodes.
</ResponseField>

### parent

<ResponseField name="parent" type="DataItem<IForceDirectedDataItem>">
  Data item of a parent node.
</ResponseField>

### x

<ResponseField name="x" type="number | undefined">
  X coordinate for fixed positioning.
  
  When set, the node will be locked to this position.
</ResponseField>

### y

<ResponseField name="y" type="number | undefined">
  Y coordinate for fixed positioning.
  
  When set, the node will be locked to this position.
</ResponseField>

### value

<ResponseField name="value" type="number">
  Value of the node as set in data.
</ResponseField>

### sum

<ResponseField name="sum" type="number">
  Sum of child values.
</ResponseField>

### valuePercentTotal

<ResponseField name="valuePercentTotal" type="number">
  Percent value of the node, based on total sum of all nodes in upper level.
</ResponseField>

### valuePercent

<ResponseField name="valuePercent" type="number">
  Percent value of the node, based on the value of its direct parent.
  
  **Since:** 5.2.21
</ResponseField>

### category

<ResponseField name="category" type="string">
  Category name of the node.
</ResponseField>

### depth

<ResponseField name="depth" type="number">
  Node's depth within the hierarchy (0 for root).
</ResponseField>

### node

<ResponseField name="node" type="LinkedHierarchyNode">
  Reference to the visual LinkedHierarchyNode element.
</ResponseField>

### circle

<ResponseField name="circle" type="Circle">
  Circle element of the node.
</ResponseField>

### outerCircle

<ResponseField name="outerCircle" type="Circle">
  Outer circle element of the node (shown for nodes with children).
</ResponseField>

### label

<ResponseField name="label" type="Label">
  Reference to node's Label element.
</ResponseField>

### fill

<ResponseField name="fill" type="Color">
  Node's auto-assigned color.
</ResponseField>

### fillPattern

<ResponseField name="fillPattern" type="Pattern">
  Node's auto-assigned pattern.
  
  **Since:** 5.10.0
</ResponseField>

### disabled

<ResponseField name="disabled" type="boolean">
  Indicates if node is currently disabled (collapsed).
</ResponseField>

### parentLink

<ResponseField name="parentLink" type="HierarchyLink">
  HierarchyLink leading to parent node.
</ResponseField>

### childLinks

<ResponseField name="childLinks" type="Array<HierarchyLink>">
  Array of HierarchyLink objects leading to child nodes.
</ResponseField>

### linkWith

<ResponseField name="linkWith" type="Array<string>">
  Array of IDs of directly linked nodes (for custom cross-links).
</ResponseField>

## Data Format

```typescript
interface IForceDirectedDataObject {
  name?: string;
  value?: number;
  children?: IForceDirectedDataObject[];
}
```

### Example Data

```javascript
forceDirected.data.setAll([{
  name: "Central Node",
  value: 0,
  children: [
    {
      name: "Branch 1",
      value: 10,
      children: [
        { name: "Leaf 1.1", value: 5 },
        { name: "Leaf 1.2", value: 3 }
      ]
    },
    {
      name: "Branch 2",
      value: 15,
      children: [
        { name: "Leaf 2.1", value: 8 },
        { name: "Leaf 2.2", value: 7 }
      ]
    }
  ]
}]);
```

### Example with Fixed Positions

```javascript
forceDirected.data.setAll([{
  name: "Center",
  value: 10,
  x: 400,  // Fixed X position
  y: 300,  // Fixed Y position
  children: [
    { name: "Node 1", value: 5 },
    { name: "Node 2", value: 8 }
  ]
}]);
```

## Methods

### restartSimulation()

```typescript

```

Restarts the force simulation with the given alpha (energy) value.

<ParamField path="alpha" type="number">
  Energy level for the simulation (0 to 1).
</ParamField>

```javascript
forceDirected.restartSimulation(0.5);
```

### updateNodePositions()

```typescript
updateNodePositions(): void
```

Updates visual node positions based on the force simulation state. Called automatically on each simulation tick.
